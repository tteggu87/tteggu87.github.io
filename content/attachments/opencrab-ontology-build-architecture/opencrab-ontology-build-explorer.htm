아래 내용은 9-Space를 도메인 해석 렌즈로 읽고 현재 OpenCrab 코드의 canonical 매핑과 구현 상태를 비교한 질적 분석이다. 상태 표시는 성능 점수나 공식 로드맵 진척률이 아니다.

9-Space는 노드 타입 목록이 아니라 도메인 해석 렌즈다

렌즈를 선택하면 어떤 질문을 던지는지, 도메인 고유 타입을 어떤 역할로 읽는지, 현재 코드에서 어떻게 매핑되는지 볼 수 있다.

MCP는 온톨로지 조회·조작의 단일 진입점이다

이 진입점을 통해 의미, 타입, 도구, 데이터, 발행의 SSOT를 일관된 방식으로 사용한다.

설계상 컴파일러, 구현상 동적 인터프리터

단계를 누르면 의도한 계약과 현재 기본 경로의 차이를 비교할 수 있다.
